Commit Chronicle

Telegram-бот для глубокого анализа активности на GitHub. Собирает коммиты через кастомный GraphQL-движок, упаковывая данные в ультра-сжатый JSON для передачи в LLM.

GraphQL Engine Production
⚙ Архитектура
Python 3.11 • python-telegram-bot • arq • Redis • SQLite • GraphQL (HTTP/2)
Telegram ──► arq Worker ──► Redis/SQLite
                │
                ▼
  GitHub API (GraphQL/REST)
  ├── TokenPool (RateLimiter)
  └── HTTP/2 Batch (20 repos)
                │
                ▼
  Compact JSON ──► Pub/Sub ──► Chat
  • GraphQL Engine: Кастомный клиент на httpx (HTTP/2) с курсорной пагинацией.
  • Aliases Batching: Сбор истории коммитов 20 репозиториев за один HTTP-запрос.
  • TokenPool: Раздельный лимитинг REST/GraphQL и контроль стоимости до запроса.
  • Compact Result: Сжатие JSON на 55–60% для LLM через ручную сериализацию.

Nexus

Событийно-ориентированное ядро управления (Control Plane), мониторинга и автоматической ИИ-диагностики (AIOps) для фермы Telegram-ботов и микросервисов.

AIOps Control Plane
⚙ Архитектура
Python 3.11 • FastAPI • aiogram 3 • Redis (Pub/Sub) • Docker API (Proxy) • Gemma 4 (AITunnel)
  GitHub Actions ──► FastAPI Webhook
                          │
                          ▼ (Redis Pub/Sub)
  Collector ──► EventBus ──► Incident ──► Gemma 4
      │ (5s)                 │
      ▼                      ▼
  Docker Containers ──► Telegram Alert
  • DooD (Docker-out-of-Docker): Изолированное и безопасное взаимодействие через docker-socket-proxy с жесткими whitelist-правилами.
  • Auto-AI Diagnostics: Мгновенный фоновый разбор логов аварий моделью Gemma 4 при возникновении сбоев.
  • State Change Detection: Локальный StateCollector опрашивает контейнеры раз в 5 секунд, генерируя точечные события изменений для защиты от спама.
  • Decoupled Webhook Receiver: Независимый сервис на FastAPI, принимающий хуки от GitHub Actions с HMAC-SHA256 подписью.
  • Incident Management: Атомарная блокировка открытия инцидентов (Redis SET NX) с защитой от дублей.
